# 要件定義書

**システム名：** りらくもみのき 予約管理システム  
**対象店舗：** りらくもみのき 富士錦町店  
**作成日：** 2026年5月  
**バージョン：** 1.0  

---

## 1. システム概要

### 1.1 目的
マッサージサロン「りらくもみのき」の予約業務をオンライン化し、お客様がスマートフォン・PCから24時間いつでも予約できる環境を提供する。あわせて、スタッフのシフト管理・売上集計・会計処理を一元管理する。

### 1.2 技術スタック
| 項目 | 内容 |
|------|------|
| フロントエンド | Next.js 14（App Router）/ TypeScript / Tailwind CSS |
| バックエンド | Next.js API Routes（サーバーレス） |
| データベース | Supabase（PostgreSQL） |
| 認証 | Supabase Auth（メール・Google・Apple） |
| メール送信 | Resend |
| デプロイ | Vercel |

## 2. 利用者と役割

### 2.1 ユーザー種別

| 種別 | 説明 | アクセス範囲 |
|------|------|------------|
| **一般ユーザー（お客様）** | 会員登録したお客様 | トップページ・予約フロー・マイ予約 |
| **スタッフ（管理者）** | `admin_users`テーブル登録済みのスタッフ | 管理画面全般（報酬管理以外） |
| **オーナー** | `admin_users.role = 'owner'` | 管理画面全般＋報酬管理 |
| **未ログインユーザー** | 未認証の訪問者 | トップページのみ閲覧可 |

## 3. お客様向け機能要件

### 3.1 トップページ（`/`）

- 店舗名・営業時間・最終受付時刻を表示（データベースの設定値から動的取得）
- メニューをカテゴリー別に一覧表示（「オプション」カテゴリーは非表示）
- 各カテゴリーに最安値・所要時間範囲を表示
- ログイン済みの場合：本日の予約（確定済み）を表示
- 「ご予約はこちら」ボタンで予約フローへ遷移

### 3.2 会員登録・ログイン（`/auth/login`）

- メールアドレス＋パスワードによる認証
- Googleアカウントによるソーシャルログイン
- Appleアカウントによるソーシャルログイン
- ログイン後：リダイレクト元のページへ自動遷移

### 3.3 予約フロー（`/reserve`）

ウィザード形式で5ステップ。ログイン必須。

#### ステップ1：日付選択
- カレンダーから予約日を選択
- 選択可能範囲：本日〜3ヶ月後
- 過去日・4ヶ月以上先は選択不可

#### ステップ2：コース選択
- カテゴリータブで絞り込み可能
- 各コースのメニュー名・所要時間・税込価格を表示
- `duration_minutes > 0` のメニューのみ表示（オプションを除外）

#### ステップ3：スタッフ選択
- 「おまかせ（指名なし）」または特定スタッフを指名
- 指名料：¥1,650（固定）
- 選択した日に休みのスタッフはグレーアウト・選択不可

#### ステップ4：時間選択
- 現在時刻を表示（30秒ごとに自動更新）
- **「今すぐ予約」ボタン**：最短で案内できる時間を自動表示・1タップで選択
- **「時間を指定する」ボタン**：午前・午後・夜間のグループ別に10分単位で選択
- 予約不可のスロット（締切超過・満席）は表示しない

#### ステップ5：確認・予約確定
- 予約内容サマリー（日付・時間・コース・スタッフ・合計金額）を表示
- **支払い方法選択**：「店頭現金払い」または「ネット決済（準備中）」
- 顧客情報入力（お名前：必須、電話番号：任意）
- 「予約を確定する」ボタンで予約完了

#### 完了画面
- 予約番号の表示
- 予約内容の最終確認
- 前日にリマインドメールが届く旨の案内

### 3.4 マイ予約（`/my-reservations`）

- 今後の予約・過去の予約を一覧表示
- 各予約のステータスをバッジ表示（予約確定・キャンセル・来店済み）
- **キャンセル機能**：来店1時間前まで可能

### 3.5 メール通知

| タイミング | 内容 |
|----------|------|
| 予約確定時 | 予約内容・日時・担当スタッフ・金額の確認メール |
| 前日20時 | 翌日の予約リマインドメール（Vercel Cronで自動送信） |

## 4. 管理者向け機能要件

### 4.1 管理画面ナビゲーション（`/admin/*`）

サイドバーにSVGアイコン付きのメニュー。アクセス制御：
- スタッフ・オーナー共通：スケジュール・シフト・分析・メニュー・スタッフ・設定
- オーナー専用：報酬管理

### 4.2 予約管理表（`/admin`）

- 日付ナビゲーションで前後の日付に移動
- **1日サマリー**：本日の予約件数・会計済み・未会計・本日売上（税込）
- **スタッフ別グリッド**（タイムライン形式）：
  - スタッフ名列は横スクロール時も固定表示
  - 出勤・勤務外・休みをカラーで視覚的に区別
  - 予約ブロックをクリック → 編集モーダル
  - グリッドの空きエリアをクリック → 手動予約入力（スタッフ・時刻が自動入力）
  - 「会計」ボタンで会計処理モーダル
- PDF出力：サイドバーなしの印刷専用ページ（`/print`）

### 4.3 手動予約入力

- グリッドの空きエリアクリック、または「手動予約追加」ボタン
- 入力項目：スタッフ・メニュー・日付・時間・顧客名・性別・備考・予約経路（電話/来店）
- 1日1予約制限・締切時間チェックなし（管理者権限のため）

### 4.4 会計処理

- 「会計」ボタンから会計モーダルを開く
- オプションの追加・変更
- 会計確定 → ステータスを `completed` に更新

### 4.5 シフト管理（`/admin/shifts`）

月間カレンダー形式で各スタッフのシフトを管理。  
優先順位：日別上書き ＞ 休み申告 ＞ 週間定期シフト ＞ デフォルト（営業時間全体）

- 週間定期シフトの設定（曜日ごとの出勤時間）
- 日別上書き（特定の日だけ出勤時間を変更）
- 休み申告（特定の日を休みに設定）

### 4.6 集計・分析（`/admin/analytics`）

- 期間フィルター（日・週・月・年）
- 売上推移グラフ
- 予約件数推移グラフ
- スタッフ別売上・予約ランキング
- CSVダウンロード

### 4.7 メニュー管理（`/admin/menus`）

| 項目 | 内容 |
|------|------|
| 操作 | 追加・編集・削除 |
| 管理項目 | メニュー名・カテゴリー・所要時間（分）・価格（税込）・表示順・有効/無効 |

### 4.8 スタッフ管理（`/admin/staff`）

| 項目 | 内容 |
|------|------|
| 操作 | 追加・編集・削除 |
| 管理項目 | スタッフ名・表示順・有効/無効・報酬設定 |

### 4.9 設定（`/admin/settings`）

| 設定キー | デフォルト値 | 説明 |
|---------|-----------|------|
| `business_start_time` | 10:00 | 営業開始時刻 |
| `last_checkin_time` | 23:00 | 最終受付時刻 |
| `cutoff_minutes_before` | 60 | 予約締切（分前） |
| `reservation_slot_minutes` | 10 | 予約枠の間隔（分） |
| `total_beds` | 5 | 同時施術可能ベッド数 |

### 4.10 報酬管理（`/admin/compensation`）オーナー専用

- スタッフごとの給与・報酬計算
- 売上連動の報酬管理

## 5. ビジネスルール・制約

### 5.1 予約制約

| ルール | 詳細 |
|--------|------|
| 1日1予約制限 | 同一ユーザーは1日1回のみ予約可能（管理者入力は除く） |
| 予約締切 | 予約開始時刻の60分前まで（設定で変更可） |
| キャンセル締切 | 来店1時間前まで |
| 予約可能期間 | 本日〜3ヶ月後 |
| 同時予約上限 | 同時間帯の予約数がベッド数（デフォルト5）を超えない |

### 5.2 料金ルール

| 項目 | 金額 |
|------|------|
| 指名料 | ¥1,650（固定・税込） |
| メニュー料金 | メニューごとに設定 |
| 合計 | メニュー料金 ＋ 指名料（指名時のみ） |

### 5.3 スタッフ割当ロジック（おまかせ時）

1. 当日出勤中のスタッフを取得
2. 指定時間帯が空いているスタッフを検索（sort_order順）
3. 最初に見つかったスタッフを自動割当
4. 空きスタッフがいない場合はエラー

### 5.4 支払い方法

| 種別 | 状態 |
|------|------|
| 店頭現金払い | 実装済み（デフォルト） |
| ネット決済 | UI実装済み、決済連携は将来対応（Stripe等） |

## 7. 非機能要件

### 7.1 パフォーマンス
- 予約フロー各ステップの画面表示：3秒以内
- 空き時間APIのレスポンス：2秒以内

### 7.2 セキュリティ（将来対応予定）
- Supabase RLS（行レベルセキュリティ）による自分のデータのみアクセス可
- APIレート制限（DoS攻撃対策）
- セキュリティヘッダーの設定（XSS・クリックジャッキング対策）
- 環境変数のVercel環境変数への移行（.env.localから）

### 7.3 対応デバイス
- スマートフォン（iOS / Android）：お客様向け画面はモバイルファースト
- PC / タブレット：管理画面はPCでの使用を想定
